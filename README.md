# ai_auto_marketing

主打两个动作：`批量爬取微信文章`、`智能提取群二维码`，并把采集结果进一步整理后`推送到 GitHub`。

这是一个把搜索、筛选、生成文案、批量触达串成一条链的多平台自动营销原型，尤其适合需要持续沉淀微信私域线索和内容资产的场景。

当前重点能力：

- `批量爬取微信文章`
- `智能提取微信群二维码`
- `结果整理后推送 GitHub`
- `Tardis 营销专用一键工作流` ⭐NEW

当前已接入：

- `X.com`
- `哔哩哔哩`
- `领英` ⚠️ 当前不可用（反爬机制严格，搜索功能暂失效）
- `微信` 公众号文章爬虫

适合想快速验证这类流程的人：

- 多账号 Cookie 登录管理
- 关键词搜内容
- 同一关键词并行搜索 `X.com`、`哔哩哔哩` 和 `领英`
- 基于产品信息自动生成营销文案
- 批量回复/评论
- 一键执行"先搜索再评论"的双平台工作流
- Tardis 批量营销（遍历 40+ 关键词，每个关键词搜索 100 条、评论 100 条）
- 评论自动强制带链接（硬约束）
- 记录成功、失败、平台、时间和错误原因

## 为什么这个项目容易让人点进来

这不是一个只会"发模板话术"的静态页面，而是一套真正跑得起来的营销工作台：

- 一个后台，管理多个平台账号
- 一个产品库，统一驱动不同平台文案
- 一个搜索页，先找热点内容再触达
- 一个回复任务页，直接批量执行
- 一个统计页，回看哪些策略和平台更有效

核心目标很直接：

> 少做重复操作，把"找内容 -> 写文案 -> 发出去 -> 看结果"这件事压缩到几分钟内完成。

## 功能亮点

- `多平台账号管理`
  保存 Cookie，验证登录状态，默认选中最近登录账号

- `指定用户帖子批量评论` ⭐NEW
  针对特定用户的帖子进行批量营销推广，获取指定用户的最近 N 条帖子并自动评论。支持 X.com 和哔哩哔哩。

- `微信公众号文章爬虫` ⭐NEW
  使用 Playwright 绕过反爬机制，批量爬取微信文章，智能提取群二维码（多重验证 + 图像识别），支持关键词搜索、多页翻页和实时进度显示

- `搜狗微信爬虫优化` ⭐UPDATED
  - 修复翻页功能，现在可以正确遍历多页结果
  - 改用「标题+关键词」组合去重，避免同一文章重复存储
  - 自动检测并跳过无效链接

- `Tardis 营销专用一键工作流` ⭐NEW
  - 针对 Tardis 数据服务的高意图关键词搜索
  - 自动过滤 Sober 用户，排除 airdrop/giveaway/meme 等噪音帖子
  - 4 种评论策略：诊断型、价值型、样例型、演示型
  - 每条评论强制带 Tardis 链接（硬约束）
  - 支持批量营销：遍历 40+ 关键词，每个关键词搜索 100 条、评论 100 条
  - 支持 X.com、哔哩哔哩全平台

- `评论风控保护` ⭐NEW
  - 哔哩哔哩评论时自动检测「CD 时间未到」等风控提示
  - 触发风控时自动停止批量任务，避免持续失败
  - 评论成功后自动等待 2 秒，降低被限流概率
  - 剩余任务保留，下次可继续执行

- `GitHub 推送`
  可将采集得到的文章、二维码识别结果和相关数据整理后推送到 GitHub，方便归档、协作和二次处理

- `内容搜索`
  按关键词抓取平台内容，保留标题、链接、点赞、评论等指标

- `双平台并行工作流`
  输入一次关键词，同时调用 `X.com` 和 `哔哩哔哩` 在线账号并行执行"先搜索，再批量评论"

- `自动生成营销文案`
  根据产品信息生成 `soft / medium / hard` 三种回复策略

- `自动附带宣传链接`
  从产品描述中提取推广链接，自动拼接到回复内容

- `批量执行回复`
  统一调度账号、内容和产品，直接批量跑任务

- `默认值优化`
  默认选中最近设置的产品，搜索数量默认 `100`，最大评论数默认 `100`

- `失败可追踪`
  页面会显示失败原因，便于快速排查是权限、风控、页面结构还是内容本身的问题

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 准备环境

需要本机已安装：

- `Python 3.13+`
- `Google Chrome`
- `chromedriver`，默认路径 `/usr/bin/chromedriver`

### 3. 启动项目

```bash
python app.py
```

浏览器打开：

```text
http://127.0.0.1:5055
```

## 使用流程

### 1. 添加平台账号

进入 `/accounts`：

- `X.com` 账号 Cookie 至少需要 `auth_token`
- `哔哩哔哩` 账号 Cookie 至少需要 `SESSDATA`

添加后可直接点击"验证 Cookie"。

### 3. 配置产品

进入 `/products` 填写产品名称、描述、价格、推广信息。

当前逻辑会自动从以下字段提取宣传链接：

- `description`
- `wechat_id`

### 4. 搜索内容

进入 `/search`：

- 选择账号
- 输入关键词
- 搜索对应平台内容

系统会保存：

- 标题或正文
- 原始链接
- 点赞/评论等指标
- 平台信息

页面也支持一键清空：

- `X.com` 已保存帖子
- `哔哩哔哩` 已保存帖子
- 全部已保存帖子

### 5. 双平台一键工作流

进入 `/search` 的 `X.com / 哔哩哔哩 一键工作流`：

- 输入一个关键词
- 选择产品
- 设置搜索数量、最大评论数、策略

系统会自动：

- 找到最近在线的 `X.com` 账号
- 找到最近在线的 `哔哩哔哩` 账号
- 两个平台并行执行"先搜索，再批量评论"
- 某个平台没有在线账号时自动跳过

## 当前定位

这是一个"可运行、可改造、可继续扩展"的自动化营销原型，不是承诺稳定成功率的生产系统。

现实限制需要直接讲清楚：

- 平台页面结构变化会影响自动化
- 不同账号的评论/回复权限不完全一致
- 内容越硬广，越容易被平台限制
- Cookie 过期后需要重新提取
- 部分页面可浏览但不一定允许当前账号发言

## 数据存储

默认使用本地 SQLite：

```text
marketing.db
```

已支持按平台隔离：

- 账号记录带 `platform`
- 搜索结果带 `platform`
- 回复记录带 `platform`
