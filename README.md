# ai_auto_marketing

把搜索、筛选、生成文案、批量触达串成一条链的多平台自动营销原型。

当前已接入：

- `X.com`
- `哔哩哔哩`
- `Telegram` 群组营销

适合想快速验证这类流程的人：

- 多账号 Cookie 登录管理
- 关键词搜内容
- 同一关键词并行搜索 `X.com` 和 `哔哩哔哩`
- 基于产品信息自动生成营销文案
- 批量回复/评论
- 一键执行"先搜索再评论"的双平台工作流
- 记录成功、失败、平台、时间和错误原因

## 为什么这个项目容易让人点进来

这不是一个只会"发模板话术"的静态页面，而是一套真正跑得起来的营销工作台：

- 一个后台，管理多个平台账号
- 一个产品库，统一驱动不同平台文案
- 一个搜索页，先找热点内容再触达
- 一个回复任务页，直接批量执行
- 一个统计页，回看哪些策略和平台更有效

核心目标很直接：

> 少做重复操作，把"找内容 -> 写文案 -> 发出去 -> 看结果"这件事压缩到几分钟内完成。

## 功能亮点

- `多平台账号管理`
  保存 Cookie，验证登录状态，默认选中最近登录账号

- `Telegram 群组营销`
  批量搜索 Telegram 群组、自动发送营销消息、黑名单过滤（自动跳过 Sober/戒酒等关键词群组）

- `内容搜索`
  按关键词抓取平台内容，保留标题、链接、点赞、评论等指标

- `双平台并行工作流`
  输入一次关键词，同时调用 `X.com` 和 `哔哩哔哩` 在线账号并行执行"先搜索，再批量评论"

- `自动生成营销文案`
  根据产品信息生成 `soft / medium / hard` 三种回复策略

- `自动附带宣传链接`
  从产品描述中提取推广链接，自动拼接到回复内容

- `批量执行回复`
  统一调度账号、内容和产品，直接批量跑任务

- `默认值优化`
  默认选中最近设置的产品，搜索数量默认 `100`，最大评论数默认 `100`

- `失败可追踪`
  页面会显示失败原因，便于快速排查是权限、风控、页面结构还是内容本身的问题

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 准备环境

需要本机已安装：

- `Python 3.13+`
- `Google Chrome`
- `chromedriver`，默认路径 `/usr/bin/chromedriver`

### 3. 启动项目

```bash
python app.py
```

浏览器打开：

```text
http://127.0.0.1:5055
```

## 使用流程

### 1. 添加平台账号

进入 `/accounts`：

- `X.com` 账号 Cookie 至少需要 `auth_token`
- `哔哩哔哩` 账号 Cookie 至少需要 `SESSDATA`

添加后可直接点击"验证 Cookie"。

### 2. 配置 Telegram Bot（可选）

进入 `/telegram`：

1. **创建 Bot**：在 Telegram 搜索 `@BotFather`，发送 `/newbot` 创建机器人
2. **获取 Token**：创建成功后会收到 Bot Token，格式如 `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`
3. **填写 Token**：在页面输入 Token，点击"验证 Cookie"

**验证结果说明**：

| 状态 | 说明 |
|------|------|
| 成功 | 显示 Bot 名称（如 `hhchhchhcbot`），表示连接正常 |
| 失败 | 显示错误信息，常见原因：<br>- `SOCKSHTTPSConnectionPool...SSL: UNEXPECTED_EOF` → 代理端口错误<br>- `Max retries exceeded` → 网络问题或代理不可用<br>- `Connection refused` → 代理未启动或端口错误 |

**代理配置**：

代码会按以下优先级自动检测代理：

1. **环境变量**（最高优先级）：`HTTP_PROXY`、`HTTPS_PROXY`、`http_proxy`、`https_proxy`
2. **自动检测**：依次尝试本地常用端口 7897, 7890, 1080, 8080

如果验证失败，可以手动设置环境变量：

```bash
# 手动指定代理端口
export HTTP_PROXY=http://127.0.0.1:7897
export HTTPS_PROXY=http://127.0.0.1:7897
python app.py
```

**重试策略**：

代码内置了自动重试机制来处理临时网络问题：
- 最多重试 3 次
- 重试间隔递增（1秒、2秒、3秒）
- 针对 429、500、502、503、504 状态码自动重试

### 3. 配置产品

进入 `/products` 填写产品名称、描述、价格、推广信息。

当前逻辑会自动从以下字段提取宣传链接：

- `description`
- `wechat_id`

### 4. 搜索内容

进入 `/search`：

- 选择账号
- 输入关键词
- 搜索对应平台内容

系统会保存：

- 标题或正文
- 原始链接
- 点赞/评论等指标
- 平台信息

页面也支持一键清空：

- `X.com` 已保存帖子
- `哔哩哔哩` 已保存帖子
- 全部已保存帖子

### 5. 双平台一键工作流

进入 `/search` 的 `X.com / 哔哩哔哩 一键工作流`：

- 输入一个关键词
- 选择产品
- 设置搜索数量、最大评论数、策略

系统会自动：

- 找到最近在线的 `X.com` 账号
- 找到最近在线的 `哔哩哔哩` 账号
- 两个平台并行执行"先搜索，再批量评论"
- 某个平台没有在线账号时自动跳过

### 6. Telegram 群组营销

进入 `/telegram`：

1. **搜索群组**：输入关键词搜索 Telegram 群组
2. **发送营销**：选择产品、设置营销内容，批量发送到群组
3. **黑名单过滤**：系统会自动跳过群组名称含有 `sober`、`戒酒`、`戒毒`、`康复` 等关键词的群组

## 当前定位

这是一个"可运行、可改造、可继续扩展"的自动化营销原型，不是承诺稳定成功率的生产系统。

现实限制需要直接讲清楚：

- 平台页面结构变化会影响自动化
- 不同账号的评论/回复权限不完全一致
- 内容越硬广，越容易被平台限制
- Cookie 过期后需要重新提取
- 部分页面可浏览但不一定允许当前账号发言

## 数据存储

```text

```

已支持按平台隔离：

- 账号记录带 `platform`
- 搜索结果带 `platform`
- 回复记录带 `platform`
